# ADR-0009: Structured File Surgical Merge

## Status

Accepted (2026-04-29)

## Context

`sync.sh` において、JSON や YAML などの構造化ファイルを同期する際、従来は「全上書き」か「Pin（同期対象外）」の二択しかなかった。これにより、リポジトリ固有のカスタマイズ（例：独自の scripts や ignore パターン）を維持しつつ、共通の推奨設定を強制することが困難であった。

## Decision

`yq` を活用した「外科的マージ（Surgical Merge）」機能を導入する。

### 仕様

1. **対象ファイル**: `.json`, `.yaml`, `.yml` 拡張子を持つファイル（`lefthook.yaml` 等の一部例外を除く）。
2. **動作**:
    - `sync.sh -y`（自動モード）: 対象ファイルがターゲット側に存在する場合、`yq` を使用して `dist/` の内容をターゲット側にマージする。
    - 対話モード: 差分がある場合、従来の `[y/N/pin/all]` に加え、`m` (merge) オプションを提供し、選択時にマージを実行する。
3. **マージロジック**: `yq eval-all 'select(fileIndex == 0) * select(fileIndex == 1)' target dist` を使用。
    - `dist/` に存在するキーはターゲット側の値を上書き（強制適用）。
    - ターゲット側にのみ存在するキーは維持（個別カスタマイズの保護）。
    - 再帰的にマージされる。

### 依存関係

- `yq` (mikefarah/yq) v4 以上がインストールされている必要がある。

## Consequences

### Easier

- `package.json` や `biome.json` などの設定ファイルにおいて、共通規約を強制しつつ、プロジェクト固有の設定を共存させることができる。
- 「Pin」による完全な同期停止を避け、継続的に推奨設定のアップデートを受け取れるようになる。

### Harder

- `yq` への依存が発生する。
- マージによってコメントが消失したり、フォーマットが崩れたりする可能性がある（必要に応じて各言語のフォーマッターで事後処理することを推奨）。
