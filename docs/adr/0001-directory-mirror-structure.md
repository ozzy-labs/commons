# ADR-0001: Directory Mirror Structure for Sync

## Status

Accepted (2026-04-04)

## Context

dev-config は共有ファイルとテンプレートを各リポジトリにコピーする。コピー元とコピー先のマッピング方法を決める必要がある。

選択肢:

1. **マニフェストファイル** — YAML で src/dest を個別定義
2. **命名規則** — ファイル名やサフィックスでコピー先を判定
3. **ディレクトリミラー** — ソースのディレクトリ構造がそのままコピー先になる

## Decision

ディレクトリミラー方式を採用する。`shared/` と `templates/` の内容を対象リポジトリのルートにそのままコピーする。

## Consequences

### Easier

- コピー先がディレクトリ構造から自明。マッピング定義が不要
- ファイルの追加・削除時にマニフェスト等の更新が不要
- 新しいエージェント（`.codex/` 等）の追加がディレクトリを置くだけで完了

### Harder

- ソースとコピー先で異なるパスにしたい場合は対応できない（現時点でその要件はない）
