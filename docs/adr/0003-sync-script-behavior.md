# ADR-0003: Sync Script Behavior

## Status

Accepted (2026-04-04)

## Context

sync.sh が共有ファイルをコピーする際、既存ファイルとの差分がある場合の振る舞いを決める必要がある。

選択肢:

1. **無条件上書き** — 常に上書き。シンプルだが意図しない変更のリスク
2. **差分表示 + 確認** — 変更があるファイルを一覧表示し、上書き/キャンセルを選択
3. **ファイル単位の選択** — 各ファイルについて個別に上書き/スキップを選択

## Decision

差分表示 + 確認方式（選択肢 2）を採用する。

sync.sh の動作:

1. 共有ファイル: コピー対象一覧とコンフリクトの有無を表示し、「すべて上書き」または「キャンセル」を選択
2. テンプレート: 対象ファイルが存在しなければコピー、存在すればスキップ（確認不要）

ファイル単位の選択（選択肢 3）は不採用。共有ファイルは全リポジトリで同一であるべきなので、一部だけ更新する運用は想定しない。

## Consequences

### Easier

- 意図しない上書きを防止できる
- 差分がなければ確認なしで完了（ノイズが少ない）
- テンプレートは自動判定（存在チェック）で確認不要

### Harder

- 完全自動化（CI からの実行等）には `--force` オプションが必要
